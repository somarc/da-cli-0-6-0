f005 · route · fixed

route classify labeled live code assets as orphan

Symptom

Classifying a real, served code asset — da route classify /scripts/aem.js, which returns 200 on the code bus — reported orphan. The classifier inferred ownership only from content sourceLocation and ignored helix status code.status: 200 already in the same status payload.

Why it matters

Orphan cleanup and route audits become dangerous if healthy code-bus paths look ownerless.

Resolution

Fold code.status (and code.sourceLocation) into ownership. No DA source + code-bus presence ⇒ codebus. Result also surfaces code status for agents.

Lock

Unit tests and live smoke on /scripts/aem.js.
