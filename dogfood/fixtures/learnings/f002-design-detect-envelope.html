f002 · design · open

design detect gave no answer on a design-less site

Symptom

Running da design detect against a site with no design system exited with an error and printed nothing to standard output.

Why it matters

There is a real difference between “the command failed” and “there is no design system here,” and an agent cannot tell them apart from an empty error. Wave 2 design surfaces need structured orientation either way.

Resolution

Planned: return a structured result for the no-design-system case — a clean envelope that says so — rather than an empty error. This is the principle the whole release is built on: every operation should leave the agent better oriented, including when the answer is “nothing found.”

Lock

Pending: unit test for structured no-design-system envelope (ADR 0001). Until fixed, agents should treat empty stderr+exit 1 as ambiguous and fall back to design audit / design rules.
