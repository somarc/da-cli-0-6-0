f002 · design · fixed

design detect gave no answer on a design-less site

Symptom

Running da design detect against a page with no anti-patterns left empty stdout (only a stderr info line). Agents could not tell “clean scan” from “command failed.” Related: token-check for a site without Stardust brand tokens exited 1 without a clear status name for “no design system here.”

Why it matters

There is a real difference between “the command failed” and “nothing to report.” Wave 2 design surfaces need structured orientation either way — the whole release principle: every operation leaves the agent better oriented, including when the answer is “clean” or “no tokens.”

Resolution

Fixed in da-cli (src/lib/design-detect.js + design command):

  • design detect always emits an envelope: ok, status: clean | findings, summary, findings, next[]
  • Clean scans are exit 0 with non-empty JSON — not silence
  • design token-check emits status: tokens-present | no-design-tokens with missing/present lists and recovery hints

Lock

Unit tests in design-detect.test.js. Smoke: da --format json design detect / → status: clean; token-check / → status: no-design-tokens on sites without Stardust tokens. 0.6.0 friction gate: High — closed.
