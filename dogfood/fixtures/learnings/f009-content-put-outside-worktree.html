f009 · content · fixed

content put refuses files outside the worktree

Symptom

Agent workflows often materialize HTML or binaries under /tmp/… then run:

da content put /index.html /tmp/index.html --commit

da-cli refuses: the local file is outside the current workspace.

Why it matters

Default refuse is correct safety — agents must not upload arbitrary machine paths. The product debt was orientation: the agent needs a clear “you are outside the workspace; if you mean this, pass the flag and own the harm.”

Resolution

Policy unchanged: refuse out-of-tree by default. Contract improved:

  • Message names the workspace boundary and the intentional flag --allow-outside-worktree
  • Ownership language: passing the flag means you accept risk (secrets, wrong files, out-of-repo artifacts)
  • Prefer staging under the project (fixtures / content-staging/) then put without the flag
  • Error code CONTENT_OUTSIDE_WORKTREE plus next[] / recovery hints for agents

Not done (and not required for this fix): auto-allow /tmp, silent auto-stage, or removing the guard.

Lock

Unit tests in content.test.js assert message shape, ownership tone, error code, and allow path. 0.6.0 friction gate: High — closed on orientation (policy was already right).
