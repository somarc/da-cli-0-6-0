f009 · content · open

content put refuses files outside the worktree

Symptom

Agent workflows often materialize HTML or binaries under /tmp/… then run:

da content put /index.html /tmp/index.html --commit

da-cli refuses with a hard error: local file is outside the current working tree; pass --allow-outside-worktree if intentional.

Why it matters

Good safety for humans; high friction for agents. The recovery flag exists but is easy to miss when the error is unexpected. Agents that always stage under the repo avoid it; agents that use OS temp dirs loop on failure.

Workaround (today)

  1. Write under the repo (e.g. dogfood/fixtures/… or content-staging/…), then put, or
  2. Pass --allow-outside-worktree when the temp path is intentional.

Resolution (planned)

Keep the default refuse (safe). Improve the error envelope: name the flag, suggest a repo-relative staging path, and surface next[] commands. Optional: project config allowOutsideWorktree: true for known agent sandboxes. Lock with tests for refuse + allow paths.

Lock

Pending product/docs fix. Observed during dual-site content puts from /tmp in dogfood sessions.
