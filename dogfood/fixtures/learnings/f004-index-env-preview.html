f004 · index · fixed

index query --env preview silently queried live

Symptom

While exercising the index surface, da index query default --env preview returned live results, not preview. The command had kept a local --env alias for --target, but root --env already means DA Admin API environment (prod|stage|dev). The value was swallowed before the index command could see it, defaulting to live with a plausible URL.

Why it matters

Same trust class as f001: silent wrong target with a successful-looking response trains agents to stop verifying.

Resolution

Drop the index-local --env alias entirely. Index validate/query resolve the host only via --target preview|live. If root --env is mistakenly set to preview or live, the CLI hard-errors with recovery text pointing at --target.

Lock

Unit tests on resolveIndexQueryHost.
