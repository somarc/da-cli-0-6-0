f003 · index · fixed

Query index could not be built through da

Symptom

Wave 3 makes coverage index-driven: each page declares wave, surface, and status as metadata. The index is a projection of the live site — preview never updates it. The tool could only read the index; agents fell back to raw curl against admin.hlx.page/index/….

Why it matters

If the certifying surface requires leaving the CLI, the dogfood has failed its own rule: prove the surface with da.

Resolution

da index build POSTs the Helix admin index endpoint (* or a path), requires root --commit, and returns structured next steps. Index show/validate/query emit a published-state caveat; publish page reminds agents to reindex after live promote.

Lock

CLI primitive + published-state caveats on read paths. Public proof: this site’s query-index.json after build.
