f008 · content · open

content merge 404s on a DA folder path

Symptom

After da content clone --path /data, the natural follow-up da content merge /data threw:

DaApiError: DA API 404 at https://admin.da.live/source/{org}/{repo}/data

Merging the workspace with no path (da content merge) succeeded and reported “Merged 2 file(s)”.

Why it matters

Agents chain clone → merge → diff → put-tree. A folder path that works for clone should not 404 on merge, or the CLI should reject the path with a structured recovery (“merge the workspace root, not a DA folder URL”). Empty 404 bodies leave the agent guessing.

Workaround (today)

  1. da content clone --path /data --force (or --all)
  2. da content merge with no path (or path that the workspace merge API accepts)
  3. da content diff then put/push as needed

Resolution (planned)

Either (a) make merge <folder> resolve to workspace-relative merge of that subtree, or (b) hard-error with an ADR-0001 envelope: wrong path shape, next commands, and no silent 404. Lock with a unit/integration test.

Lock

Pending product fix. Wave 1 evidence: dogfood/evidence/wave-1/content-merge-data.txt (404) vs content-merge.txt (success).
