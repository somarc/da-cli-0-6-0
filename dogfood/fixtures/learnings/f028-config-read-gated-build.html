f028 · index, auth · fixed

index build died on a permission it didn't need — the verification read gated the build

Symptom

Wave 5 drill C3, live site: index build --wait failed with Permission denied for *. But the identity was fully authorized to rebuild the index — a raw POST admin.hlx.page/index/…/* with the same token returned 202, job created. The 403 actually came from the Configuration Service read that --wait performs first to discover the verification target. An unauthorized convenience gated an authorized mutation, and the error blamed the wrong resource.

Why it matters

Authorization is not uniform across admin routes: delivery-admin rights (preview, publish, reindex) and Configuration Service rights are separate grants, and agents routinely hold one without the other. A CLI that couples them turns a partial permission set into a false hard failure — and a misattributed error (for *) sends the agent down the wrong recovery path entirely. Ground truth was a live response, not the CLI's claim — the f021 lesson again.

Resolution

Fixed in da-cli: the config read under --wait is now best-effort — on failure the command warns with the actual denied route and verifies against the default /query-index.json target; the build always dispatches. Companion fix: permission-denied messages now name the route that 403ed (denied at …/config/…), so a caller-supplied target can never hide the real denial. Field proof: the same command went from Permission denied for * to built-and-verified with one honest warning, and after the f029 recovery detected the index change total 39→40.

Lock

Hermetic CLI locks in index-build-f028.test.js: one stub plays Configuration Service (403), helix-admin index route (202 + job), and the public index host — via the new DA_PUBLIC_HOST_URL seam alongside f026's DA_HELIX_ADMIN_URL. Asserts the rebuild dispatches despite the config 403, the envelope carries the fallback warning naming the denied route, and verification completes against the default target. Critical case verified failing on old code; suite 602/602.
