audit contracts misses autoblock-only surfaces

Symptom

da audit contracts --prefix / --verify-code lists block classes found in page markup. The modal pattern is mostly link-driven (/modals/… → JS openModal), so modal often never appears as a decorated block class on the page. Contracts can be green while /blocks/modal/modal.js is unlisted — agents may think modal is out of scope.

Why it matters

Wave 2 gates use contracts as the code-bus checklist. Autoblocks (modal, fragment rewrite, widgets) are first-class PE and need the same verify-code signal.

Workaround (today)

  1. Curl or da code status the JS/CSS paths for autoblock assets.
  2. Keep a kitchen-sink link that exercises the autoblock.
  3. Document autoblocks in BLOCK-COVERAGE.md as Required even when contracts omit them.

Resolution (planned)

Extend contracts (or a sibling report) to include known autoblock modules from code bus + static analysis of scripts.js imports / /modals/ links / /widgets/ links. Structured list: page blocks vs autoblocks vs missing assets.

Lock

Pending product. Observed on kitchen-sink after modal port: modal.js/css 200 on live; contracts inventory omitted modal.
