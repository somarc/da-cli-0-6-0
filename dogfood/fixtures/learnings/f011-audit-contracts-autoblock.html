f011 · audit · fixed

audit contracts misses autoblock-only surfaces

Symptom

da audit contracts --prefix / --verify-code listed only block classes found in page markup. Link-driven patterns (/modals/… → openModal, /widgets/, /fragments/) never appeared as decorated block classes, so modal (and peers) could be green on disk while unlisted in the contracts inventory.

Why it matters

Wave 2 gates use contracts as the code-bus checklist. Autoblocks are first-class PE — agents must see the same verify-code signal as authored table blocks.

Resolution

Fixed in da-cli:

  • Inventory scans plain HTML for /modals/, /widgets/, /fragments/ link patterns → modal, widget, fragment
  • Also reads scripts/scripts.js for autoblock wiring
  • Each block row includes source: page | autoblock | page+autoblock
  • Report top-level autoblocks: [] list; verify-code probes JS/CSS for those modules too

Lock

Unit tests in agent-diagnostics.test.js / audit.test.js. Live evidence: dogfood/evidence/wave-2/audit-contracts.json includes modal, fragment, widget. 0.6.0 friction gate: closed.
