f034 · workspace, safety · fixed

the recovery instruction destroyed the work it was protecting

Symptom

Drill B1 (source drift after clone) proved the conflict guard beautifully: a push over remote drift refused with the path and reason named. Then the drill followed the tool's own recovery — "Run da content merge to pull the remote copy and re-apply your change" — and merge overwrote the modified local file with the remote copy. The workspace keeps no content snapshots (commits store paths, not text), so there was nothing left to re-apply. The guard protected the remote side of the conflict and the guided recovery deleted the local side. The drill survived only because the operator had manually copied the file first.

Why it matters

An optimistic-concurrency guard exists precisely because two parties changed the same document. A recovery path that keeps one side and silently discards the other is not recovery — it is data loss with good manners. For an agent, the trap is exact: it parses the refusal, executes the recommended command verbatim, and its work is gone. The machine channel made it worse: the drift refusal was stderr prose while stdout carried a bare planned-paths array that said nothing about any conflict.

Resolution

Fixed in da-cli: content merge now preserves any local copy that was modified since clone (and differs from the incoming remote) under .da/merge-backups/<stamp>/… before converging — named on stderr, in the envelope's data.preserved, and in warnings, with executable next (content diff, then push). The push drift refusal emits one ok:false envelope naming each drifted path. Same pass: content status emits an envelope under json (was prose when clean, bare array when dirty), clone --path accepts document paths (was a silent Cloned 0 files success), and the out-of-worktree put refusal (f009's guard) emits its envelope (was empty stdout — f032 class).

Lock

Four locks in content-workspace.test.js: modified-local + drift → backup saved with the exact local text and named in the result; unmodified local → clean converge, nothing preserved; remote-deleted + modified local → backup before removal; document-path clone. All verified failing on old code; 618/618. Field proof on the live drill page: re-injected drift → push refusal envelope → merge preserved local rep4 edit at the named backup path → diff + push per the envelope's next.
