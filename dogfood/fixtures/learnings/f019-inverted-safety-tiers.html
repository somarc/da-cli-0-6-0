f019 · publish, safety · fixed

Live-facing commands had weaker ambiguous-target protection than preview

Symptom

Found by the release-review subagent audit (safety lane): preview page refused a committed run against a config-derived target without explicit --org/--repo or --yes — but publish page, unpublish, and deploy page, whose blast radius is the live CDN, carried no such check. index build opted out entirely, and its /* wildcard defeats the bulk-path heuristic by construction.

Why it matters

Safety tiers must rank by blast radius. When the safest operation carries the strongest gate, the gate placement was incidental, not designed — an agent in the wrong directory with a stale .da.json could publish to a live site it never intended to touch, while a mere preview would have been refused.

Resolution

Fixed in da-cli: publish page, unpublish, deploy page, and index build all raised to the preview safety tier (+--yes): committed runs against config-derived targets refuse with retry guidance. Invariant stated: live blast radius is never less protected than preview.

Lock

First CLI-level publish.test.js — refusal and dry-run cases for all four commands. This lock also produced its own finding: the first push hung CI for 30s per test because the spawned CLI hit the interactive auth helper before the safety gate. That became f020 — the flywheel eating its own output.
