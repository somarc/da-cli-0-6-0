f007 · route · fixed

route clean did not remove the orphan

Symptom

Running da route clean on an orphan used to skip when no DA source remained, and only attempted a DA editor preview flush — never helix unpreview or unpublish. Live and preview stayed 200.

Why it matters

Detecting orphans without the ability to clean them leaves the certifying site (and real customer sites) permanently dirty.

Resolution

Clean always runs helix unpreview (DELETE preview) and unpublish (DELETE live), even with no DA source. Source delete runs only when daSource is true. HTTP 401/403 returns a structured permission-refused envelope with recovery steps.

Lock

Unit tests lock step order and permission paths.
