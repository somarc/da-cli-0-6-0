f018 · content, safety · fixed

content diff path traversal read arbitrary local files

Symptom

Found by the release-review subagent audit (safety lane), live-reproduced: a DA path like ../../../etc/hosts passed to content diff resolved outside .da/workspace/ and read the local file — the workspace mapping (toLocalPath) joined paths without checking the result stayed inside the workspace root.

Why it matters

In an agent-operated CLI this is an exfiltration vector: a hostile page name or a manipulated instruction gets an agent to diff a "DA path" that is actually ~/.ssh or a token cache, and the contents land in the agent's context or output. Local tooling gets attacked through its inputs, same as servers.

Resolution

Fixed in da-cli: toLocalPath refuses any DA path whose resolved local target escapes .da/workspace/ — covering leading traversal, embedded .. segments, and the merge path that shares the mapping.

Lock

Traversal and embedded-segment tests in the content workspace suite; the guard sits in the shared mapping primitive, so every current and future consumer inherits it.
