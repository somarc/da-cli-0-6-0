f010 · content · fixed

Relative /media/… image URLs rewrite to about:error

Symptom

DA HTML authored with host-less image paths:

<img src="/media/example.webp" alt="">

After preview, .plain.html showed <img src="about:error"> even when the binary existed on the content bus. Absolute content-bus URLs worked:

https://content.da.live/{org}/{repo}/media/example.webp

Why it matters

Agents default to site-relative media paths. Failures are silent in the browser (broken image). Platform media rewrite needs a fetchable absolute URL for hashing into Media Bus.

Resolution

CLI (da-cli, this release track):

  • content put / put-tree warn when HTML has host-less /media on img/source src or srcset
  • --rewrite-media-urls expands them to https://content.da.live/{org}/{repo}/media/… before upload
  • --strict-media-urls fails the put until fixed (CI / agent gates)
  • <a href="/media/…"> left alone (file/video links often work relatively)

Still platform-side: Helix could resolve same-tenant host-less /media/* itself; until then absolute URLs (or rewrite flag) are the contract. Skills should keep absolute-URL defaults.

Lock

Unit tests in media-urls.test.js. Preflight notes include f010 + recovery. 0.6.0 friction gate: High — closed on CLI warn/rewrite/strict; platform auto-resolve remains optional follow-on.
