f013 · route · fixed

route classify probes miss codebus static HTML under /tools/

Symptom

During the Wave 3 cut, da route classify /tools/wave-2-cli-surface.html returned ownership orphan with preview, live, and code probes all reporting 404 — while a plain curl of the same path on the live host returns 200. The page is static HTML checked into git under tools/ and served straight from the code bus; it is code-owned in practice, but the classifier cannot see it.

Why it matters

Orphan is the clean-candidate class. A probe gap that mislabels a living code-bus page as orphan invites route clean toward content that is very much owned — the exact wrong-orientation failure the ownership matrix exists to prevent. Agents must be able to trust that orphan means "no owner," not "the probe didn't know where to look." Extensioned static routes (/tools/*.html, and similar) are a common EDS pattern for sidekick plugins and site tooling, so this is not an exotic corner.

Interim guardrail

Documented on the route matrix at the Wave 3 cut: when class is orphan but the path is known codebus static, prefer curl/live proof and do not treat the route as a clean candidate. A live 200 that disagrees with an all-404 probe is a probe conflict, not an orphan. This guardrail remains useful for older CLI versions.

Resolution

Fixed in da-cli commit 1954c73. Before an extensioned route can be called orphan, the classifier now re-probes the literal path that the code bus actually keys. A literal code 200 promotes the verdict to codebus. Any 403 from the canonical or literal status probes now yields probe-failed: permission-denied means ownership is unknown, never absent, so route clean refuses instead of passing through its orphan safety gate. Release review upgraded the finding from Medium to High because the old result could orient an agent toward deletion without --force.

Lock

Seven focused locks in src/commands/route.test.js cover literal .html codebus success, a genuine literal 404 orphan, canonical and literal 403 handling, the extensionless no-reprobe path, and unchanged DA-content ownership. The original field command was re-run: /tools/wave-2-cli-surface.html now reports codebus with literalProbe.code: 200 (the command's documented codebus exit is 3). Before/after evidence is checked in under dogfood/evidence/wave-3/.
