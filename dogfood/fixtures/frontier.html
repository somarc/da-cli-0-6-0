What Frontier proves

  • The page proves its own claims. Every capability named on this page is exercised by this page, and audited live in the matrix below — evidence, not assertion. Same ethos as this whole site: living documentation.
  • Poster-first shaders are EDS-safe. LCP is a static image; WebGL is a post-paint enhancement that fails soft in every direction.

The scrollbar is a timeline

Keep scrolling — vertical travel drives the rail horizontally. There is no JavaScript in that motion.

01 / VIEW-TIMELINE

This rail has no scroll listeners

The tall wrapper declares view-timeline-name: --sr-scroll; the track animates with animation-timeline: --sr-scroll. The mapping is declarative and runs off the main thread — it cannot jank.

02 / ANIMATION-RANGE

Choreographed by grammar

animation-range: contain 0% contain 100% pins the slide to exactly the slice of scroll where the stage is pinned. The hairline above these cards fills on the very same timeline.

03 / FAIL SOFT

Static path complete

No animation-timeline support — or prefers-reduced-motion — and these same cards render as a plain readable stack. Feature-detect, then enhance; never the reverse.

04 / OKLCH

One hue, whole system

Every accent on these plates derives from one registered custom property: oklch(0.74 0.19 var(--frontier-hue)), with siblings at hue − 100 and hue + 70. Change one number, reskin the lab.

05 / STATUS

Where it stands, 2026

Scroll-driven animations shipped first in Chromium, landed in WebKit's 2025 release train, and Firefox's rollout is converging. Progressive enhancement — like this rail — is the sensible pattern while engines align.

How to author

In DA, a frontier-hero block is one row with two cells: copy left, poster image right, plus optional config rows (hue, preset). A scroll-rail block is one row per card. A vf-specimen block is one row of specimen text plus an optional voice row — the axis registry and font loading live in block code. A feature-matrix block is empty — or lists css / js rows to filter the probe registry. Authors edit content; the shaders and probes live in block code.
