WAVE 5 · FAILURE INJECTION · RECOVERY

When it breaks, does it say so?

Wave 5 injects failures on purpose. The proof line: each failure must be recognized (a structured envelope naming it), contained (no damage done), and recovered (an executable recovery, verified against the contract). Retrying until a command exits zero is not recovery evidence. So far every single drill has surfaced a real finding — f027 through f033 — all fixed or recovered same-session.

Opened 2026-07-16 — IN PROGRESS · Evidence: dogfood/evidence/wave-5/ · Drill matrix: dogfood/WAVES.md · Related: test plan · learnings

What Wave 5 is (plain language)

Waves 1–4 proved the happy paths and the durable paths. Wave 5 proves the CLI is honest when things go wrong — because agents act on what commands say, a failure that reports success is worse than a crash. Injection families (ADR 0002 D5):

  1. A — Orientation failures: unresolved target, expired auth, mutation without --commit.
  2. B — Content failures: source drift after clone, invalid section shape, missing block assets.
  3. C — Platform-state failures: stale preview/live, orphan routes, frozen or absent index.
  4. D — Long-running-work failures: interrupted jobs, cancellation, pipeline abort, partial batch failure.

  • f031 — the freshness tool was structurally blind to the very staleness it existed to detect.
  • f032 — safety refusals emitted empty stdout under json; now every refusal is one envelope.
