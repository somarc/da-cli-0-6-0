Learnings

This is the growing history of the dogfood run: every field problem that taught the CLI something, turned into a primitive fix, and locked with a test. The suite does not only prove commands — it records how trust was earned.

New learnings get a stable id (fNNN) and their own route under /learnings/. This hub stays the index; primary nav does not grow a link per finding.

The flywheel

Dogfooding earns its keep by surfacing real problems that unit tests cannot. Discipline:

  1. Operate a real surface with da against this site.
  2. Record the symptom, command, and wrong orientation on a learning page.
  3. Fix the primitive in the CLI (not a one-off script).
  4. Lock with a regression test (and public proof where relevant).

See also: test plan, coverage catalog, git board dogfood/CONTENT-IA.md.
