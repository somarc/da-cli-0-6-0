Wave 3 is not about pretty blocks. It is about structured data (sheets + query index) and route ownership — contentbus vs codebus vs orphan — so agents know where to edit, what to clean, and what the public index actually contains.

Cut 2026-07-14 03:05 UTC · Evidence: dogfood/evidence/wave-3/ · Related: coverage sheet · query-index.json · test plan

What Wave 3 is (plain language)

Wave 1 proved put/preview/publish. Wave 2 proved blocks and audits. Wave 3 proves the map of the site:

  1. Sheets — DA JSON tables discovered by da content sheets (strict :type: sheet + data[] only — inventory, not full multi-sheet model; expand as awareness grows).
  2. Query index — live /query-index.json built from published pages + Configuration Service content/query.yaml; exercised via da index show / validate / query / build.
  3. Route ownership — for each path, is it contentbus, codebus, hybrid, orphan, or probe-failed? Commands: route classify / audit / canonical / clean.

Without this wave, agents guess wrong about “edit DA vs edit git” and ship broken reindexes.
