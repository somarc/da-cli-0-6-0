TRUST · ORIENTATION · PROOF

Why build the proof this way

An agent-first CLI is only useful if an agent returns to it for the next decision instead of reaching for raw APIs. That happens when the tool is trustworthy — and trust is not a claim you make in a README. It is something you demonstrate, repeatedly, against reality.

So the guiding principle for da-cli is blunt: every operation should leave the agent better oriented than before it ran. Know what you touched, what it proved, what is safe next, and what needs a human. This site is built to prove the tool lives up to that.

The trust arc

Three commitments build on each other, each closing a gap where an agent would otherwise have to take something on faith.

  • Trust the contract. Every operation — success or failure — returns the same machine-readable shape, so an agent parses one envelope and always knows what comes next. A failure is structured data with a recovery action, not prose on stderr.
  • Trust correctness. A command that "succeeds" must have caused an observable state transition — DA source changed, preview activated, the public URL renders the expected bytes. A zero exit code is never enough.
  • Trust provenance. This site was provably constructed by the recorded da commands, verifiable down to a live URL, and regenerable from a checked-in pipeline. You do not have to believe a release note; you can replay it.

Where this sits in the ecosystem

Honesty about position is part of the trust arc. da-cli is not Adobe canon. The canonical developer tool for Edge Delivery is the aem CLI (helix-cli), and this project is not a replacement for it or a contest with it — it is an independent build with a different center of gravity: agent-first, full-gamut operation across the DA Admin API, the Helix Admin API, and public delivery, with structured envelopes, dry-run safety, route ownership, audits, durable jobs, and failure recovery as first-class primitives.

Where the canonical tooling and this one disagree about an edge — probe semantics, path grammar, merge behavior — the discipline here is to name the edge as a numbered learning, fix what is ours to fix, and carry the rest downstream as a concrete, evidenced conversation rather than a claim. The learnings history is that record. A tool earns its place in an ecosystem the same way it earns an agent's trust: by demonstration.

And the ecosystem is moving toward this thesis, not away from it. AI authoring surfaces are arriving on DA itself, and AI personalization is arriving at the delivery edge — of1, early-stage Adobe AEM technology, adapts a page per visitor. Both ends of the pipeline are becoming machine-operated. That makes the middle — a headless loop that proves what the canonical bus actually serves, audits what machines author, and recovers when it drifts — more necessary, not less. When every visitor may see a different render, the proof contract stays scoped to what can be certified: DA source → preview → live, the canonical response for a route. Personalization is presentation downstream of that proof, and machine-readable provenance (of1 pages already carry Stardust provenance markers, the same skill da stardust wraps) is where the trust arc extends next.

Why a whole site — and why it is not plain

One rendered page cannot exercise content trees, structured sheets, route ownership, durable jobs, migration, code sync, index configuration, and — most importantly — failure recovery. The dogfood site is a small but complete Edge Delivery site precisely so that every meaningful surface of the CLI has somewhere real to act: kitchen-sink blocks, learnings flywheel, nav chrome, media bus, worker stubs, the construction pipeline that regenerates what you are reading.

Early drafts of this page said the site was deliberately plain — that design was not what da-cli is for. That was a useful humility when the only proof we had was structure. It is no longer true of this place.

What you see now is intentional craft on the same rails: Renderaissance paper and ink, a lit painterly home hero, Dualform PE in the labs and gutters, a classical Cupid mark in the header, industry-scale block spectrum on the kitchen sink. That magnificence is not marketing varnish on top of the dogfood. It is part of the proof surface — media paths, absolute content-bus URLs, block contracts, codebus CSS, authored chrome — all put, previewed, and published through da.

Beauty does not replace the trust arc. A gorgeous page that cannot be regenerated from construct.yaml is still untrusted. A regenerable site that also holds its own as an EDS specimen is the standard we hang the hat on: structure, correctness, and provenance first — and craft that agents can actually operate. The prose still carries the story. The tool still carries the proof. The site no longer pretends those require beige.
