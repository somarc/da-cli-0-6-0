Coverage

This is the living table of contents for the test suite. The top navigation stays deliberately small — the narrative spine of the site — while every surface a wave exercises is catalogued here. As the suite grows, this page grows; the navigation does not.

The scalable target is to generate this list from page metadata through the query index: each test page declares its wave, surface, and status; the index aggregates them; a tagged page appears here automatically. Until then this page is curated by hand — and curating it is itself part of exercising the content surface.

Wave 1 — Foundation and core loop

  • Home — the site's purpose and the release rule.
  • Why — the trust arc: contract, correctness, provenance.
  • Test plan — the waves and the release gate.
  • Surfaces exercised: config, status, content put/get/list, preview page, site info, fstab mount, pipeline run.

Wave 2 — Blocks and audits

  • Blocks and the proof surface — built from the columns and cards blocks; proves block code resolves on codebus.
  • Surfaces exercised: audit contracts --verify-code, audit full, block inspect, design detect.

Wave 3 — Structured data and routes

  • Query index — every page above declares its wave, surface, and status as metadata; the published site is indexed into query-index.json and read back with da index show / validate / query. This is the structured spine a future block will render this table from.
  • DA sheet — an authored coverage data sheet classifies every command family by wave and required/optional class, discovered with da content sheets.
  • Route ownership — da route audit maps the whole site; da route classify was exercised across content-owned, code, and orphaned routes, including a deliberately orphaned page to drive route clean.
  • Surfaces exercised: content sheets, index show/validate/query, route classify/canonical/audit/clean.
  • Findings surfaced: #3 (index build), #4 (index --env), #5–#7 (route classify/clean) — the flywheel turning.
